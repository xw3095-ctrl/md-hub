# 回测框架设计

## 设计原则

1. **模块化**：策略逻辑与数据分离
2. **可复现**：随机种子、版本控制
3. **免存活偏差**：使用上市满一定时间的数据
4. **交易成本**：考虑滑点和佣金

## 架构设计

```
回测引擎
├── DataHandler    → 数据接入层
├── Strategy       → 策略逻辑层
├── Portfolio      → 组合管理
├── RiskManager    → 风控模块
└── Performance    → 绩效分析
```

## 关键指标

| 指标 | 计算公式 | 说明 |
|------|----------|------|
| 年化收益率 | $(1+R)^{252/N} - 1$ | 年化 |
| 夏普比率 | $(R_p - R_f) / \sigma_p$ | 风险调整收益 |
| 最大回撤 | $\max(\text{peak} - \text{trough})$ | 尾部风险 |
| Calmar 比率 | 年化收益 / 最大回撤 | 收益/回撤比 |
| 胜率 | 盈利交易 / 总交易 | 交易质量 |

## 常见陷阱

- 🔴 **前视偏差 (Look-ahead Bias)**
- 🔴 **过分优化 (Overfitting)**
- 🔴 **忽略交易成本**
- 🔴 **幸存者偏差 (Survivorship Bias)**
